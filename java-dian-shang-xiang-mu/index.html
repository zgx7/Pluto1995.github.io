Cors 跨域问题

原理：
	要求ajax 请求必须在同IP 同端口 同协议下进行，不然就会产生跨域问题
    
解决方案：
	方案一： Controller 添加 @CrossOrigin 注解，但是只有是牵扯到跨域问题 就需要单独添加这个注解，那么 有没有一劳永逸的方法呢？ 那就是方案二！
	方案二：在网关服务添加 跨域的配置类 CorsConfig， 核心是给Spring容器中添加一个bean对象CorsWebFilter，通过配置这个对象，实现 跨域访问 方法访问 请求头 携带cookie 等属性的配置
	 @Bean
    public CorsWebFilter corsWebFilter() {

        // 配置cors 跨域配置对象
        CorsConfiguration configuration = new CorsConfiguration();
        configuration.addAllowedOrigin("*");
        configuration.addAllowedMethod("*");
        configuration.addAllowedHeader("*");
        configuration.setAllowCredentials(true);


        // 配置源对象
        UrlBasedCorsConfigurationSource configurationSource = new UrlBasedCorsConfigurationSource();
        configurationSource.registerCorsConfiguration("/**", configuration);

        // cors 过滤对象
        return new CorsWebFilter(configurationSource);
    }
