Java集合

一、*ArrayList

	ArrayList  底层是通过  Object[] 来进行数据存储， 顺序结构，长度固定，相对简单 [初始胡数组长度为10，不足10，取10]
自动扩容机制:
	添加元素前，会进行检查添加元素后，原数组的长度是否能够完全容纳 添加元素，不满足就会触发 扩容机制  grow(); 扩容到原数组长度的1.5倍大小。

常用方法：
	基本上也是操作底层数组，长度不够时，会创建新长度的数组，进行重新指向；删除元素时，会进行元素前移，末尾置空

二、*LinkedList

LinkedList：
	由于同时实现了 List 和Queue 和 Deque[双向队列]  所以具有三者的通用方法
	底层主要由三个属性：firstNode  lastNode  itemData； 
	Node<E> node；节点类，node中封装了 上一个节点prev  下一个节点next 和自己携带的数据 element,每个节点通过上下两个节点相互引用进行绑定，双向引用感觉结构比较稳定。
	由于是双向绑定，在add remove peek get [CRUD] 操作时，需要双向解绑，在重新双向绑定新节点，将需要删除的元素置空。
	当一个对象在没有元素引用时，会被GC进行回收
	
	
需要注意的：
	由于链表结构存在头、尾结构，所以在处理这两个位置时，需要多思考，还有LinkedList中first和last 引用节点的问题，没有操作这里，可能会导致GC无法回收 或者回收错误。
	node 节点数据也可以存放null 和 Object  而两者在进行比较时，用的方式不一样  [null == ]，[Object.equals()]

三、ArrayDeque

ArrayDeque:
	跟ArrayList 一样，底层是一个Object[] ,不同的是，初始化长度是16，不允许存放null值，根据head tail 进行首位判断，head不常等于0，
	巧妙的地方在于 add() 添加 核心是 'head = (head-1)&(element.length)' 这一个计算公式，它通过当head为正数时，取余实现head=head-1，即head前移一位；当head等于-1时[只有-1这种情况]，进行取模操作，即等于下标最大值，相当于进行首位相连；【tail操作也是同理】，当head==tail时，会进行扩容；
	
ArrayDeque的扩容机制；
	不同于ArrayList 的扩容机制，ArrayListDeque是进行两倍扩容，触发扩容时，创建一个两倍原数组长度的新数组，然后head右边的值顺序复制到新数组，在复制head左边的 按照下标大小从小到大复制

其他：
	ArrayDeque在添加或者通过构造函数进行长度较长的集合转换时，会计算和判断ArrayDeque的可扩容长度；
	private void allocateElements(int numElements) {
        int initialCapacity = MIN_INITIAL_CAPACITY;
        // Find the best power of two to hold elements.
        // Tests "<=" because arrays aren't kept full.
        if (numElements >= initialCapacity) {
            initialCapacity = numElements;
            initialCapacity |= (initialCapacity >>>  1);
            initialCapacity |= (initialCapacity >>>  2);
            initialCapacity |= (initialCapacity >>>  4);
            initialCapacity |= (initialCapacity >>>  8);
            initialCapacity |= (initialCapacity >>> 16);
            initialCapacity++;

            if (initialCapacity < 0)   // Too many elements, must back off
                initialCapacity >>>= 1;// Good luck allocating 2 ^ 30 elements
        }
        elements = new Object[initialCapacity];
    }
这段代码是比较由意思的：当你输入一个数[int类型]，它来判断距离你这个数下一个2的N次方的数是多少；
    原理： 当你输入一个数，通过‘|’运算，从这个数为1的最高位，开始向右进行 全部变为1操作，因为最高位16位数[int],所以当你这个数从为1的最高位开始向右都给你变成1，然后在+1 就得到你这个数下一个2的N次方的数了

四、HashMap和HashSet

【HashMap】：
	Java7——底层是数组+链表实现：初始化数组长度为16；[一开始是空，HashMap的空参构造器只指定了增长因子，没有指定数组长度的大小，第一次put添加元素时，会检查底层数组是否为空，然后在进行初始化]
	put方法：第一次检查并初始化，长度16，然后这里我们把数组长度看作N[因为要牵扯到扩容，这里的n值为2的幂次，为什么呢，方便-1后，低位全是1，然后通过(n-1)‘&’num操作，将num截取到只在0-（n-1）以内],添加的key 会计算hash值，通过取余得到hash，然后数组查找这个hash，得到tab[i],就是这个链表，在然后遍历这个链表有没有对应的key，有就覆盖，没有就在后面新增[Java8，Java7在前面新增]；
【值得注意的是】HashMap中的链表是单向的。这里需要区别  [Node<K,V>]只有nextNode  没有prevNode
	
【数据扩容】：数组在到达阈值时，[与扩容因子有关]在扩容时两倍扩容，链表个数超过8时，会变成红黑树结构。
	这里主要说一下数组扩容后，数据的前移，即数据所在数组下标是否需要变化。原来数据的hash需要重新计算
【个人理解】： 
	因为数组每次扩容到原来长度的两倍，在二进制中相当于向左移动一位，就是开放了一个高位，如果重新计算的hash高位是0，那么就下标就不需要改变，数据也不需要迁移。如果是1，就需要迁移数据。那么 如何迁移数据呢？
	【数据迁移】，跟新增类似，根据重新的hash值进行匹配。

【HashSet】
	底层是对HashMap进行的封装，相关的方法是合适的HashMap方法进行调用实现。
	其中，HashSet的值对应HashMap的key，存储的value是new Object[]

五、LinkedHashMap和LinkedHashSet

【前要】
	LinkedHashSet也是对LinkedHashMap的一种封装，所以我们主要说一下LinkendHashMap
【LinkedHashMap】
	结构：与HashMap相比较，LinkedHashMap的链表是双向的，通过一个哑元head 把所有的entries 都首尾串联起来了[通过Entry实现 而不是Node]
	新增：在冲突链头部插入新的entry元素[相当于在双向链的尾部插入]。
	优点：在查找元素时，可以通过遍历双向链来查找key值对应的数据而不需要查找hash值，找到数组的下标，在遍历冲突链。

六、 TreeMap与TreeSet

【前提】
	同样的TreeSet也是对TreeMap进行的一层封装，来实现的。
【底层】
	通过红黑树来实现，添加元素时，可能会进行 旋转 染色 重新引用 等操作，但是红黑树的 要求时 高树的高度不能超过低树高度的一倍。
    
【原理】
	对红黑树的操作，增删改查 等问题。理解了红黑树 就不难理解 这部分了。

八、其他拓展

1. 如何实现线程安全 [JUC]

【写时复制 实现】
	ArrayList --> CopyOnWriteArrayList
	Set --> CopyOnWriteArraySet
原理：
	写时复制就是在写文件时，会将数据复制出来一份，操作这个副本进行数据的增删改，然后在将指针指向操作完的副本，在这一过程中，其他的读操作不受影响， 但是数据不是及时的，而且写操作是单线程，不支持并发操作，[读读并发，读写并发，写写不并发]

【Collections】
	工具类可以将集合转换成  支持同步操作。

2. 红黑树的理解

【简单理解 -- 基于 2-3-4 结构】
	2节点[一个元素]时，该节点颜色为黑色，添加一个节点时，会出现[一黑一红 看作3节点，这里有两种情况 向左还是向右 即较大的值在右下，还是较小的值在左下]，在添加一个节点时，会形成[一黑带两红 4节点形式]，即中间值会变成黑色，左边是较小值[红色]，右边是较大值[红色]， 差不多就这样。。。
	当被插入的节点是四节点时，处于中间节点的值，会与父节点组成 三节点，后面的节点会组成 2节点和3节点形式，优先减少旋转的形式。

总结

	集合是一种存储数据的容器，它通过 数组 链表 树结构等形式，来更多的保存数据，选择合适的结构有利于提高 操作数据的速度，减少复杂度。
	了解好底层如何进行方法的实现，可以更清晰的理解数据是如何在结构中操作流转，不至于黑盒操作。
